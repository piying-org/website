# 客户端介绍

> 皮影是一套跨框架的表单解决方案，通过统一的 Schema 定义，实现一次编码，多端通用。

## 为什么选择皮影？

### 一次定义，多端使用

- 传统表单：视图、逻辑、类型需要分别定义,并存在强耦合
- 皮影：一次 Schema 定义，解决默认值、验证、类型、布局等问题
- 优势：代码管理更方便，支持框架间无缝迁移

### 灵活的布局控制

- Schema 定义固定，但组件布局可以动态调整
- 不受限于固定的展示形式，完全按设计需求定制

### 完整的逻辑表达能力

- 支持 `anyOf`、`oneOf` 等复杂逻辑
- 使用 `FieldLogicGroup` 轻松实现条件切换

### 数据流分离

- 支持模型→视图、视图→模型的独立转换
- 通过 `transfomer`/v.transform 处理数据格式转换

---

## 技术特点

- **统一API**：为不同框架提供一致的开发体验
- **语义化配置**：减少开发心智负担
- **高内聚低耦合**：通过配置动态调整功能
- **Valibot 兼容**：完整支持 Valibot 的验证与转换机制

## 皮影如何支持不同平台

### 动态解析部分

必须实现部分,用来将定义动态解析为配置

### 静态解析部分

默认下动态解析后,大多数场景都可以直接使用

小部分场景因为视图和逻辑分离,所以需要使用静态解析,将定义先解析为对应模板并写入文件,再配合动态解析赋值

#### 举例

各个`小程序`便是视图/逻辑分离的,需要使用静态解析先生成模板
